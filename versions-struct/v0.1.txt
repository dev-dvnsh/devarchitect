____________________________________________________________________________

Hard field constraints---
Short fields = max 100-150 words
Lists = max 5-7 items
scores = strict numeric range
no markdown inside json
no explanations outside json


-----FINAL OUTPUT RULES---------
Output MUST be valid JSON.
No extra commentary.
Max 5 functional requirements.
Each description ≤ 120 words.
Each risk ≤ 80 words.
Each roadmap day ≤ 5 bullet points.
Complexity score 1–10 only.

THIS MAKES OUTPUT MORE STABLE


_________________________________________________________________________________
Command Structure(v0.1)
==> {devarchitect init "project idea"}
-> Saves project metadata locally
-> Generates project_summary + functional_requirements

==> {devarchitect analyze}
Generates:
  non_functional_requirements
  architecture_decision
  system_architecture
  idea_level_risks

==> {devarchitect stack}
Generates tech_stack_options
Generates deployment_architecture

==> {devarchitect roadmap}
Generates execution_roadmap
Generates complexity_analysis

==> {devarchitect export --format=md}
Exports full compiled architecture document

    config.json

WHY THIS ---
Each command writes to its own file.
User can manually edit any file.
Future commands can re-read + refine.
Easy to version control.
Easy to extend later.

------FLOW OF COMMANDS----- REQUIRED else ERRROR
1 init
2 analyze
3 stack
4 roadmap
5 export


HOW WE TALK TO LLM______________________________________________________________-
Each command:
    Has its own system prompt
    Has its own strict JSON schema
    Loads required previous state
    Injects only relevant context
    Validates output


JSON ENFORCEMENT STRATERGY AS LLM BREAKS JSON ----------------------
Step 1 — First Pass
Call LLM
Expect strict JSON
Strip:
Leading/trailing garbage
Try to unmarshal into Go struct
If success → done.
If fail →

Step 2 — Repair Pass (LLM-Assisted)
We send:
The following JSON is invalid.
Fix it strictly according to this schema.
Do not add commentary.
Return only valid JSON.

Include:
The broken JSON
The exact schema again
Try parsing again.
If still fails →

Hard error.
No silent acceptance.

v0.1 Scope — Team-Ready but Simple

We lock the scope so it’s achievable and meaningful:

Single-directory, project-level state
  .devarchitect/ inside project folder
  Multiple JSONs: summary.json, analysis.json, stack.json, roadmap.json
Strict dependency enforcement
  Commands must run in order: init → analyze → stack → roadmap → export
  Fail if prerequisites missing
LLM Modular Prompts
  Each command has its own system prompt
  Loads only required previous JSON files
  Strict JSON schema enforcement with double validation
CLI architecture
  Modular commands (init, analyze, stack, roadmap, export)
  Internal separation: cmd/, internal/llm/, internal/schema/, internal/state/, internal/formatter/
Team-safe starting point
  Repo can hold .devarchitect/
  Simple warnings if existing files will be overwritten
  Conflict detection is manual for v0.1
  No advanced locking yet
Constraints
  Max words per field (120–150)
  Lists max 5–7 items
  No commentary outside JSON
